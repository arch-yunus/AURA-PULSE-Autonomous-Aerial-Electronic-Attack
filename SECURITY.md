# Security Policy - Aura-Edge

## 1. Reporting a Vulnerability

We take the security of the Aura-Edge framework seriously. If you discover a vulnerability, please report it privately to the security team.

Please **do not** create public GitHub issues for security vulnerabilities.

### Scope
- **Flight Core (C++):** Memory safety and logic vulnerabilities.
- **Protocol Layer:** Cryptographic weaknesses in AES-256 or FHSS logic.
- **GCS:** Input sanitization and data leakage.

## 2. Security Standards
Aura-Edge adheres to the following security guidelines:
- **MISRA-C++:2008:** For flight-critical software.
- **CERT C++:** Safe coding standards.
- **NIST SP 800-38D:** For GCM mode implementation details.

## 3. Responsible Use
The Aura-Edge framework is intended for professional engineering and research applications. Unauthorized use in kinetic systems is strictly prohibited by the license agreement.
